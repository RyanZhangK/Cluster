# 模块管理规则

## 模块启动顺序

项目模块必须按以下顺序启动：

1. `database_process.py` - 数据库模块 (基础服务)
2. `mqtt_process.py` - MQTT通信模块
3. `flask_app.py` - Web服务模块
4. `game_process.py` - 游戏逻辑模块

## 模块依赖关系

- Web服务和游戏逻辑模块依赖数据库模块
- 游戏逻辑模块依赖MQTT通信模块
- 启动非数据库模块前需等待数据库初始化完成(最长60秒)

## 模块重启策略

- 每个模块最多自动重启3次
- 重启间隔至少1秒
- 数据库模块重启需优先处理

## 模块通信方式

- 模块间通过数据库共享数据
- 关键状态变更需记录日志
- 避免直接进程间通信，统一通过主进程协调

## 日志规范

- 每个模块有独立日志文件
- 日志文件按日期时间命名
- 单个日志文件最大1MB，保留3个备份
- 日志级别：
  - DEBUG: 开发调试信息
  - INFO: 正常运行信息
  - WARNING: 潜在问题
  - ERROR: 功能错误
  - CRITICAL: 系统级错误

## 异常处理

- 模块崩溃需自动重启(不超过最大次数)
- 未处理异常需记录完整堆栈信息
- 资源不足错误需立即告警
